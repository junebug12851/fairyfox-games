## v0.7.0 — 2026-07-02

Daily maintainer run. **Maintain:** no open issues/PRs; baseline 133/133 green.

- **New game — Loft** (`games/loft/`): a keepy-uppy juggle, mechanically distinct from
  every existing game. Glowing orbs fall under gravity; **tap** near a *falling* orb to
  bat it back up, and each one you catch on its way down scores. You can only strike a
  **descending** orb — so the mechanic is a rhythm (let it rise and fall, then catch it
  low), not a mash — and one tap can rescue a whole cluster. Every few points another
  orb joins the air, up to six, so a calm one-orb warm-up becomes a busy juggle. A
  dropped orb ends the run. Whole simulation is a pure core (`applyTap`, `stepOrb`,
  `orbGrounded`, `topUpOrbs`, `tick`, `lowestFalling`) with the only-falling bat rule as
  the guarded invariant. Full quality bar: core + **24 tests** (incl. a **self-play
  winnability** run) + shell + index (boot-failure fallback, guarded loop) + README +
  package.json.
- **Grew Ricochet** — **progression ranks**: reaching a score milestone now flashes a
  rank toast (Sharpshooter → Impossible geometry), pure logic in the core (`milestoneAt`)
  and range-scanned in the shell so a banked multi-target shot can't skip a rank. Gives a
  long run a sense of climb, alongside the existing per-shot chain banks. +2 tests (21 →
  23). Start-screen tip refreshed.
- Registered Loft in the landing `index.html` grid (tags **Reflex**, **Timing**) and
  footer, bumped the masthead **Games** count to **7**, and added the README "The games"
  row.
- **159/159 tests** green (was 133). Released **dev → main** as a tagged `--no-ff`
  **MINOR** via `release/0.7.0`.

## v0.6.0 — 2026-07-01

Daily maintainer run. **Maintain:** no open issues/PRs; baseline 110/110 green;
cleared a stale `next-steps.md` (its Ricochet-registration items were long done).

- **New game — Skyline** (`games/skyline/`): a stack-and-slice tower builder,
  mechanically distinct from every existing game. A slab slides above your tower; drop
  it (one control) and only the overlap with the slab below stays — the overhang is
  sliced off, so every sloppy drop narrows your target. A **flush** drop keeps the full
  width and pays double, so precision is the only way to keep climbing; a total miss
  topples the run; the slide speeds up with height. Whole simulation is a pure core
  (`drop`, `tick`, `moveCurrent`, `spawnCurrent`, `milestoneBetween`) — notably **no
  timer-driven death** (a slab only resolves on an explicit drop). Full quality bar:
  core + 18 tests + shell + index (boot-failure fallback, guarded loop) + README +
  package.json. Browser-verified (renders, drops, slices, no console errors).
- **Grew Polarity** — **clutch saves**: a match landed by a last-moment flip (within
  `CLOSE_TICKS`) is tallied by the pure core (`isClutch`, `g.clutch`) and surfaced on
  the game-over card — a measure of how much you played on the edge, with no play-field
  clutter. +6 tests (20 → 25). How-to/tip refreshed.
- Registered Skyline in the landing `index.html` grid (tags **Timing**, **Precision**)
  and footer, bumped the masthead **Games** count to **6**, and added the README "The
  games" row. Also backfilled the footer's missing **Ricochet** link.
- **133/133 tests** green (was 110). New/modified shells `node --check`-clean. Released
