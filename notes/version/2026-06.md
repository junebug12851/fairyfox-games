# Changelog — June 2026

Newest first. Plain-English, per commit. See `../version.md` for how this is kept.

## v0.3.0 — 2026-06-29

New game: **Orbit Slingshot** — a one-mechanic gravity game.

- A probe orbits a planet under Newtonian gravity. Hold one control to fire a
  prograde thrust (widening the orbit); release and gravity pulls you back. Sweep
  through targets to score; crashing into the planet or escaping off-screen ends
  the run. Beat your own score.
- Built to standard: a pure, JSDoc'd logic core (`orbit-slingshot.core.js`, no DOM)
  using semi-implicit (symplectic) Euler so orbits stay bounded, an external render
  shell (`orbit-slingshot.shell.js`) with a boot-failure fallback + guarded loop,
  and a 16-test suite (`node --test`) including a 600-tick orbit-stability check and
  the frame-one survival regression. Listed in the README + landing page.

## v0.2.0 — 2026-06-29

New game: **Echo Chamber** — a one-mechanic timing game.

- An echo ring expands from the centre of a circular chamber; catch it (click /
  tap / Space) the instant it crosses the target band. Each hit tightens the catch
  window; a mistimed press or an uncaught overrun costs one of three lives. Beat
  your own streak.
- Built to standard: a pure, JSDoc'd logic core (`echo-chamber.core.js`, no DOM),
  an external render shell (`echo-chamber.shell.js`) with a boot-failure fallback +
  guarded loop, and an 18-test suite (`node --test`) including an inclusive
  catch-tolerance boundary regression. Listed in the README + landing page.

## v0.1.1 — 2026-06-29

Ink Bloom boot fix + error handling, found by a real in-browser play-check.

- **Boot fix.** The render shell was an inline `<script type="module">` using a
  top-level `await import()`. On a normal page load it could fail to boot, leaving the
  start screen visible but the game dead (looks fine, does nothing). Moved the shell to
  an **external module** (`ink-bloom.shell.js`) loaded via `<script type="module"
  src>` — the conventional, robust structure — which boots reliably. Verified live in a
  browser (boots on clean load, steers, plays).
- **Error handling.** Added a boot-failure fallback (a classic script that shows a
  visible "couldn't load" message if the module never signals boot), a guarded render
  loop (a render error fails visibly and stops instead of spamming every frame), and
  global `error` / `unhandledrejection` logging. A dead-silent screen is no longer a
  possible failure mode.

## v0.1.0 — 2026-06-29

First release. The project is born as a mesh node.

- **Ink Bloom**, the first game: a one-mechanic "steer a growing line, eat motes,
  don't cross your own trail" canvas game. Built to standard — a pure, JSDoc'd logic
  core (`ink-bloom.core.js`) separated from the rendering shell (`index.html`), with a
  20-test suite (`node --test`) including a regression test for the frame-one
  self-collision bug that the original single-file prototype had.
- **Monorepo scaffold:** `games/<slug>/` layout, a landing `index.html` listing the
  games, root README (contributions welcome), CONTRIBUTING, MIT LICENSE, issue + PR
  templates, CI (tests on push/PR), and a Pages deploy workflow.
- **Mesh onboarding:** mesh-aware `CLAUDE.md`, the living-notes system, `VERSION`, and
  registration in the hub (`registry.yml` + the site's `_data/projects.yml`, as an
  `integrated` project pinned beside the site card).
