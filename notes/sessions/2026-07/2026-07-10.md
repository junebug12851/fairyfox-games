## SITE — the collection's own icon: favicon + social card + masthead · v0.19.6 · ✅ released

**What.** The owner dropped a new brand mark at `assets/icon.png` — a "game farm" emblem (a green
sprout rising from a white game-controller cube over farmland furrows, ringed by the fairyfox dashed
arcs) — and asked to "put it to use anywhere it's needed." Wired it into the shared Jekyll chrome in
three places, all from one self-hosted file:

- **Favicon / browser-tab icon** (`_includes/head.html`) — replaced the four hotlinked
  `https://fairyfox.io/favicon.*` links with `<link rel="icon">` + `<link rel="apple-touch-icon">`
  pointing at `{{ '/assets/icon.png' | relative_url }}`. Now self-hosted (no third-party request,
  consistent with the privacy posture) and gives the games section its own tab identity.
- **Social-share card** (`_includes/head.html`) — added Open Graph + Twitter `summary` meta
  (`og:type/site_name/title/description/image/url`, `twitter:card/title/description/image`) with the
  icon as `og:image`/`twitter:image` via `absolute_url`. Previously there were **no** OG/Twitter tags
  at all, so shared links had no preview image.
- **Masthead logo** (`index.html` + `assets/home.css`) — the design system already had a `.mast-logo`
  slot (`.mast-id` is flex, `.mast-logo` pre-styled in the vendored `styles.css` as a circular
  avatar). Added `<img class="mast-logo">` to the landing hero and a `home.css` override to a
  rounded-square `object-fit:contain` tile (72px, 56px on ≤560px) so the whole illustration reads and
  matches the game-card icon treatment, rather than being cropped to a circle. Override lives in
  `home.css`, **not** the vendored `styles.css` (which must stay clean for re-vendor).

**Deliberately NOT changed.** The **header brand logo** (top-left, `_includes/header.html`) stays the
shared `https://fairyfox.io/assets/icons/fox.png` — the fairyfox **hub** identity — per an explicit
owner instruction. The **11 standalone games** under `games/<slug>/` were left untouched (no favicon
added) to keep them self-contained / liftable.

**Verify.** `bundle exec jekyll build` clean (Liquid resolved: favicon `/fairyfox-games/assets/icon.png`,
`og:image` `https://fairyfox.io/fairyfox-games/assets/icon.png`). Landing page headless-previewed in
Chromium (Playwright) across **light + dark** themes and **desktop + mobile** — the logo reads, the
transparent corners show the theme background (no white box), no crop/overflow, clean alignment.
Collection **384/384** green (no game logic touched). Released `dev → main` by default on green
(PATCH) → tag `v0.19.6` → back-merge.
