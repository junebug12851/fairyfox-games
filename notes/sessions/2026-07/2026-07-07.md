## GROW — Symmetry onto varied structure (5th game on the pattern) → v0.19.3

**What.** Brought **Symmetry** onto the varied-structure + progression standard (the daily
GROW job's primary lever). Its spawn was a flat per-tick coin-flip (twin-or-single at a
random lane) — textureless noise. Now a run is a seeded **sequence of named cadences**
pulled from a stage-weighted pool, so no two runs share a skeleton and climbing the stages
visibly opens the pool.

**Core (`symmetry.core.js`).** Added `FORMATIONS` to CONFIG with six cadences, copied in
shape from Polarity into Symmetry's own core (self-contained, no shared runtime):

- **Mirror** (minStage 0, calm) — gentle alternating singles around mid lanes; the on-ramp.
- **Reflection** (0, calm) — a short run of twins to sweep up (the relief valve).
- **Cascade** (0, notable) — a rhythmic, tightening stream of alternating singles.
- **Weave** (1, notable) — flowing spread swings, lanes sweeping centre ↔ edge (sinusoid).
- **Split** (1, notable) — the mirror tradeoff as a *snap*: near-centre orb then a fast
  opposite-side edge orb, forcing a quick swing across the mirror.
- **Kaleidoscope** (2, notable) — the dense late crescendo: twin + snap rounds, tight gaps.

New pure fns `clampLane`, `pickFormation`, `loadFormation`, `spawnSpec`, `spawnNext`; a spec
is `{kind:'single'|'twin', side?, lane, gapMul}`, `gapMul` scaling `spawnInterval` (floored by
new `SPAWN_GAP_FLOOR`) so the per-stage speed-up still holds. `tick` pulls the next beat from
the current cadence and returns `formation` (a notable cadence's name at its head beat, else
null). Removed the now-unused `TWIN_CHANCE`; retired `spawnOrbs`.

**Shell + markup.** `#formCue` element + CSS (peripheral, honours prefers-reduced-motion);
`showForm(name)` flashes it when `tick` returns a notable cadence; hidden on start/death.
Start-panel copy now names the shifting cadence set.
