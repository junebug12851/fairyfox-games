## GROW — Arc onto varied structure (7th game on the pattern; rollout complete) · v0.19.5 · ⚠ UNRELEASED (env blocker)

**What.** Brought **Arc** onto the varied-structure + progression pattern — one of the five
remaining games with Growth-Architecture layers 1–2 only. Arc's `spawnTarget` placed each pad at a
flat random distance in the stage window, so every run was the same shapeless string of throws.
Now a run is a seeded **sequence of named "range" formations**, copied in shape from
Polarity/Symmetry into Arc's own core. (Considered **Loft** first — the roadmap's obvious
spawn-cadence fit — but Loft only *adds* ~5 orbs across a whole run (on score thresholds, not a
timer), so a formation queue would never drain; Arc's per-shot pad placement is the clean,
player-visible seam.) This **completes the aim/precision line** and leaves 4 of 11 to convert.

**How (core, `arc.core.js`).**

- Added `CONFIG.FORMATIONS` — 6 patterns as plain data (`id`, `name`, `minStage`, `notable`,
  `weight(stage)`, `build`): **Drift** (0, calm), **Ladder** (0, notable), **Bracket** (1,
  notable), **Groove** (1, notable), **Reach** (2, notable), **Fusillade** (3, notable). Builders
  are pure given `ctx.rng`, no rejection loops.
- `build(ctx)` returns `{f}` specs — `f` a 0..1 distance fraction. `spawnTarget` maps it across the
  **current** stage window `[max(dmin,hw), min(dmax, FIELD-hw)]`, so every pad stays on-field AND
  inside full-charge range (`landingX(1)=1024 > max window 960`, winnable) and the per-stage
  pad-shrink/spread still layers on top.
- `pickFormation(cfg, stage, rng, prevId)` — seeded, stage-weighted pick over the eligible pool,
  softly avoiding an immediate repeat (identical shape to the reference). `loadFormation(g)` fills
  `g.formSpecs`, marks the head spec, records `formId`/`formName`/`formNotable`.
- Refactored `spawnTarget` to pull one spec from the queue (loading a fresh formation when spent)
  and return a notable-head name cue. `reset` seeds `formSpecs:[]`/`formId:null` then places the
  opening pad (pulls the first, calm formation — frame-one-safe). `lob` now returns a `formation`
  field (notable head just placed, else null); added to the phase-guard early return too.
- Removed `MIN_TARGET_DIST`/`TARGET_TRIES` — formations govern spacing now (Groove clusters on
  purpose), so the old flat variety guard is obsolete.

**How (shell/HTML).** Added a `#formCue` element + peripheral flash CSS (reduced-motion honoured);
`arc.shell.js` shows it on `res.formation` in `onFlightEnd`, clears it on `beginRun`/`onDeath`.
Start-panel tips now name the range-formation set so a new player knows runs vary.

**Tests.** Replaced the now-obsolete `MIN_TARGET_DIST` variety test with **six** formation tests
(pool well-formed + non-decreasing minStage + ≥1 at stage 0; every builder ≥1 spec with fractions
in [0,1]; `pickFormation` stage-eligibility + determinism + widening pool; a long formation-driven
run keeps a pad on-field, reachable, queue never starving; distinct seeds → distinct pad sequences
+ same seed identical; notable-cue surfacing) plus the reworked opening-pad test. Arc **26 → 31**;
collection expected **378 → 383**.
