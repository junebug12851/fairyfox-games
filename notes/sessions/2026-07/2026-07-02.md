## Polarity core made fun — precision-combo redesign (v0.10.1)

Owner playtested the v0.10.0 build and said it was **still boring** — lost interest partway.
Correct call: the stages/badges were a scoreboard *around* a monotone loop, not fun in the
loop. Asked (AskUserQuestion) which directions to build the core toward; owner picked **all
four** (risk/reward tension, escalating variety, faster/denser, more to read). So I reworked
the loop, not the wrapper.

- **The fix:** a **late-flip multiplier**. Landing a needed flip within `CLOSE_TICKS` is a
  *precise* hit → `mult`++ (cap ×9); an early/safe flip → `mult` resets to 1; a gimme is
  neutral. Points = `mult` per gate, so `score` rewards nerve while `cleared` drives
  difficulty/stages. Real risk/reward every gate.
- **Variety/density:** `spawnGate` is now stage-patterned (higher stages alternate more,
  tighten spacing, burst); brisker base speed + closer gaps remove early dead time.
- **Feel:** big multiplier readout (heats up, pops precise, flinches on break); game-over
  card shows gates cleared + best ×. Copy rewritten to "flip late, not early".
- **Discipline held:** all logic pure in the core; test suite rewritten (**36 green**,
  collection **170 green**); previewed in Chrome (no console errors, HUD clean in landscape +
  portrait); `polarity.best` preserved. Committed to **dev**.
- **Note:** this is the honest lesson for the whole roadmap — **meta-progression can't rescue
  a dull core**. Before stamping the Growth Architecture onto the other six, each game's core
  loop needs the same "is the second-to-second decision actually tense?" pass. Rolled that
  into next-steps.

**Grew Ricochet** — added **score-progression ranks**. Ricochet already celebrated

games. Added a pure `milestoneAt(score)` (Sharpshooter · Trick shot · Bank master ·
Angle savant · Wall wizard · Impossible geometry) and wired a toast into the shell that

bank label takes precedence when both would fire on the same shot. +2 tests (21 → 23);
start-screen tip mentions the rank climb. No gameplay change.

**Shipped a new, mechanically-distinct game — Loft** (`games/loft/`): a keepy-uppy
juggle. Orbs fall under gravity; a **tap** strikes every *descending* orb within reach,
launching it up and scoring one point each — rising orbs ignore taps, which is the rule
that makes the mechanic a rhythm rather than a mash (and prevents one tap re-hitting an
orb it just launched). One tap can catch a cluster. `topUpOrbs` adds an orb every
`ADD_EVERY` (8) points up to `MAX_ORBS` (6), so the field fills as you climb; a grounded
orb ends the run. Distinct from all six existing verbs (steer / time-a-catch / thrust /
flip-match / aim-and-bounce / stack-and-slice) — this is multi-object gravity juggling.

Design highlights: the only-falling bat gate (`vy > 0`) is the guarded invariant
(`a rising orb ignores a tap`, `one tap cannot score the same orb twice`). Tuning was
validated empirically, not by eye: a **self-play winnability test** runs an unremarkable
"tap the lowest falling orb" policy for 1800 ticks and asserts survival + a healthy
score; an ad-hoc 5-seed / 60-second probe reached ~245 points ramping to the full 6
orbs, confirming the calm→panic curve is real and a competent player can sustain it.

Full quality bar — pure core (JSDoc'd, seeded RNG for spawns, hue palette per orb) + 24
tests (helpers, reset/spawn invariants, physics: gravity/side-bounce+clamp/ceiling/floor,
the batting rule, scoring + orb top-up cadence, floor death, dead-state inertness,
determinism, self-play winnability) + shell (danger-floor glow, per-orb danger flare,
catch rings + particles, strike ring under the cursor, boot-failure fallback, guarded
loop) + index + README + package.json.

Registered Loft: landing grid card (tags **Reflex**, **Timing**), footer link, masthead
